Java基础第三章，面向对象

面向对象概念

理解面向对象

  • 面向过程：强调的是功能行为，不断地使用函数，C语言
  • 面向对象：将功能封装进对象，只要找到了对象，就可以使用对象中的功能

面向对象的特点

  • 面向过程体现的是执行者，而面向对象体现的是指挥者（指挥对象做事情）
  • 面向对象是一种更符合人们思考习惯的思想，可以将项目的上千个功能转换成上百个对象

以后开发先找对象及其功能，若没有，自己创建对象并将所需功能定义到该对象中，便于以后复用。如果别人用，直接把我的.class文件给别人用就OK

面试对面向对象的理解

答上面两大点 + 面试场景中的例子

其实面试官您就是按照面向对象的思想在思考问题，您本身就可以开发项目，但是项目有可能比较大，一个人开发周期长。为了提高效率，您就需要有更多的具备着专业开发能力的人来帮助您做事情。这个人就是您所需的对象，只要调用这些对象的开发功能就可以完成项目，只要指挥他们做事情就可以了。这样大的项目就可以从复杂变简单，而面试官就可以从执行者变成指挥者，更符合我们的思考习惯。

男朋友也是对象，具有赚钱、做饭、洗衣服的功能。

面向对象

  • 开发：不断的创建对象，使用对象，指挥对象做事情。
  • 设计：管理和维护对象之间的关系。
  • 面向对象的特征：封装(encapsulation)、继承(inheritance)、多态(polymorphism)

类与对象的关系

java中描述事物通过类的形式体现，类是具体事物的抽象，概念上的定义。对象即是该类事物实实在在存在的个体。
类的意义在于描述事物

class Car{
    String clolr;    
    int number;    
    void run( ){    
        System.out.print("小汽车可以跑跑");    
    }    
}

类中的成员包括

  • 成员属性（成员变量）：对应类中的成员变量，就是数值或变量
  • 成员行为（函数）：对应类中的成员函数，又叫成员方法，实例方法

特点：变量定义在类中，在函数外面；函数没有 public 和 static 的修饰；类中可以没有主函数

类与对象的关系

  • 类：对事物的描述，需要体现属性和行为
  • 对象：该类事物创建的实例，通过对象调用具有的属性和行为

成员变量和局部变量的区别

  • 定义的位置不同

    • 成员变量定义在类中，在整个类中都可以被访问。
    • 局部变量只定义在局部范围内，如：函数内，语句内等。

  • 在内存中的位置不同

    • 成员变量存在于堆内存的对象中。
    • 局部变量存在于栈内存的方法中。

  • 生命周期不同

    • 成员变量随着对象的建立而建立，随着对象的消失而消失，
    • 局部变量随着方法的运行而出现，随着方法的弹栈而消失。

  • 初始化不同

    • 成员变量有默认初始化值。
    • 局部变量没有默认初始化值，必须初始化后才可以使用。

创建并使用对象

class Car  { //对Car这类事物进行描述    
    String color = "red";    
    int num = 4;    
    void show()    
    {    
        System.out.println(color+"."+num);    
    }    
}    

class CarDemo {    
    public static void main(String[] args)     {
        Car c = new Car();//建立对象    
        c.color = "black";//修改对象属性
        c.show();//使用对象的功能。    
    }    
}

对象内存结构



匿名对象

匿名对象是对象的简化形式

Car c = new Car( ); c.run( ); 简化成 new Car( ).run( );，可以简化书写
由图匿名对象的内存情况 可知，对匿名对象调用属性是没有意义的

匿名对象两种使用情况

  • 当对对象方法仅进行一次调用的时，匿名对象可以简化书写；但如果堆对象进行多个成员调用时，必须给对象起名字
  • 匿名对象可以作为实际参数进行传递

封装

###表现
函数、类都是封装体，（电脑主机机箱也可以理解为一个封装体）

好处

  • 隐藏了实现细节，仅对外提供可以访问的方式，便于调用者的使用。（定义）
  • 提高代码的复用性

封装原则

将不需要对外提供的内容都隐藏起来；把属性都隐藏，提供公共方法对其访问。

private关键字

  • 是一个权限修饰符；用于修饰成员(成员变量和成员函数)；被私有化的成员只在本类中有效，类外不可访问
  • 常用于类中不需要对外提供的内容（包括属性和行为）都私有化。
  • 属性全部私有化，对外提供对应的setXxx，getXxx方法对其进行访问。可以在set方法中加入逻辑判断，保证其合法性和合理性，若不合法抛出异常中断程序，提高对数据访问的安全性

构造函数

作用

在对象创建时，对对象进行初始化。new的同时初始化

何时使用构造函数？

如果对象一产生本身就需要具备一些内容，那么就需要用到构造函数

特点

  • 没有返回值类型也没有返回值，因为构造对象创建完就结束，不需要结果。void也不要写，因为要有区别于一般函数
  • 构造函数是有 return; 语句的，用于结束初始化动作的。或也可以放在逻辑判断中用于提前结束初始化动作
  • 构造函数名称和类名一致

默认构造函数

定义：只要定义一个类，该类中默认就有一个空参数的构造函数，是编译器编译时添加到class文件中的。但是如果在类中自定义了构造函数，那么默认的空参数的构造函数编译器就不会自动添加。

原理：没有定义对象的初始化过程，编译器会添加一个默认的初始化过程；如果定义了指定对象的初始化过程，默认的就不添加了。

构造函数的细节

  • 一个类中可以有多个构造函数，他们是以重载的方式存在的（函数名相同，参数列表不同）
  • 构造函数是有return语句的，用于结束初始化动作的。或也可以放在逻辑判断中用于提前结束初始化动作
  • 构造函数是能被private修饰的，这样该类的对象只能在本类中创建，其他程序new一个对象时会报错无法访问
  • 一般函数不能调用构造函数，但是构造函数可以调用一般函数

构造函数和一般函数的区别

运行上

  • 构造函数：对象一创建就会自动调用相应的构造函数
  • 一般函数：对象创建后，手动调用所需的一般函数

调用次数

  • 构造函数：在对象创建时仅调用一次（初始化动作只做一次而且先执行）
  • 一般方法：可以被对象调用多次

名称

  • 构造函数和类名相同，类名的书写规范是首字母大写
  • 一般函数命名规范是首字母小写，第二个单词往后单词首字母大写

构造函数之间的调用

class Person{        
    private int age;    
    private String name;    

    private Person(int a) {    
        age = a;    
    }    

    Person(String n, int a) {    
        this(a);    //调用Person(int a)构造函数    
        name = n;    
    }    
}
  • this(实参列表);就会调用对应的构造函数，而不能像一般方法那样直接写函数名调用
  • 调用其它构造函数的语句必须定义在构造函数的第一行，否则会报错，因为初始话动作要先执行
  • 一个构造函数中不能调用两个构造函数，和第二行原则冲突
  • 两个构造函数不能互相调用，会陷入死循环，一直调用，栈内存只进不出，导致栈内存溢出

this关键字

特点：this代表其所在函数所属对象的引用。哪个对象调用了this所在的函数，this就代表了哪个对象

this原理：this关键字来记录住对象的地址，并通过this来明确被初始化的对象。

作用

  • 当在函数内需要用到调用该函数的对象时，就用this。
  • 用this标识哪个变量是成员变量，此时this可以省略。但是当局部变量和成员变量同名时，必须用this表示成员变量。


题：静态方法中不能用this，因为静态方法直接被类调用，而this指代的是对象

static关键字

引入

创建对象就是为了产生实例，并进行数据的封装。而调用功能时，却没有用到这些对象中封装的数据，虽然可以编译并运行，但是该对象的创建没有意义，并且会浪费堆内存中的空间。举例来说：若类中有一个函数 

public void sleep( ){ print("zzz"); }

该函数没有调用成员变量。那么当创建多个实例的时候，new仍会在对内存中开辟空间，但是开辟的空间存储的成员变量没有被使用。

修饰函数

这是一个成员修饰符（成员变量和成员函数），被static修饰的方法除了可以被对象调用外，还可以被类名直接调用，此时没有new对象，也就不会在堆内存中开辟空间，存放成员变量。

什么时候使用静态变量？如果功能不需要访问来中定义的成员变量（非静态）时，该功能就需要静态修饰。虽然不用静态修饰也可以编译运行，但是浪费堆内存空间

静态修饰方法

静态方法只能访问静态成员，但是非静态方法即可以访问静态成员，又能访问非静态成员。

静态方法中不允许出现 this，super 关键字

静态方法可以被对象调用，但是很业余。专业的做法就是用类调用。因此静态方法也叫类方法

原理

  • 静态是随着类的加载就加载了，随着类的消失而消失
  • 静态优先于对象存在，被对象共享。
  • 因为静态先存在于内存中，无法访问后来的对象中的数据，所以静态无法访问非静态，而且内部无法书写this，因为这时对象有可能不存在，this没有任何指向

静态修饰变量

该成员变量的值，每个对象都有且一致时，就对该成员变量进行静态修饰

与成员变量的区别

  • 所属范围不同：静态变量所属于类，成员变量所属于对象。因此静态变量又叫类变量，成员变量也叫实例变量
  • 调用不同：静态变量可以被对象和类调用（一般都用类名调用）；成员变量只能被对象调用
  • 加载时期不同：静态变量随类加载而加载，成员变量随着对象加载
  • 存储区域不同：静态变量存储在方法区中，成员变量存储在堆内存中。

代码块

静态代码块

  • 写在类中 static { }
  • 随着类的加载而执行，因为类仅加载一次，所以静态代码块只执行一次
  • 用于给类进行初始化
  • 静态代码块在静态变量显示初始化之后再执行

构造代码块

  • 写在类中 { }
  • 只要创建对象，就会被调用，给所有对象初始化。而构造函数只给对应的对象（对应参数列表）初始化
  • 作用：里面可以定义不同构造函数的共性代码

局部代码块

  • 写在方法中 { }
  • 作用：可以控制局部变量的生命周期，用完直接释放掉

静态的主函数

主函数是静态的，因此要想被主函数直接调用，函数必须是静态的。

若函数不是静态的，只能在主函数中创建一个函数所在类的对象，通过对象调用非静态函数

public static void main(String [] args)

    public： 权限足够大
    static：不需要对象，直接用给定的类名就可访问：Demo.main
    void：虚拟机调用主函数，虚拟机不需要main的主函数
    main：函数名，该名字是固定的
    String [] args：主函数的参数列表：字符串属性类型的参数
    args：arguments：变量名，可以自定义，只不过人们约定俗成就这么用

被static修饰后的特点

随着类的加载而加载；优先于对象存在；被所有对象所共享；可以直接被类名调用

使用注意

静态方法只能访问静态成员；静态方法中不可以写this，super关键字；主函数是静态的

对象的创建过程

加载Demo.class文件进方法区，并进行内存分配

如果有静态变量先默认初始化，再显示初始化（赋值），如果有静态代码块，要执行仅一次

通过new，在堆内存中开辟空间，并明确首地址值，对对象中的属性进行默认初始化

调用对应的构造函数进行初始化

  • 调用父类构造函数super（）
  • 成员变量的显示初始化（给默认初始化赋值）
  • 构造代码块初始化 { }
  • 构造函数内的自定义初始化

对象初始化完毕后，将地址赋值给d引用变量

单例设计模式

概念

解决某一类问题行之有效的解决办法，强调思想而非代码。业界有23种设计模式，我们学的是其中的单例（Singleton）设计模式

单例设计模式

解决的问题？保证一个类的对象唯一性

使用场景：多个程序都要使用一个配置文件种的数据，且要实现数据共享和交换。这就必须要将这多个数据封装到一个对象中，且多个程序操作的是同一个对象，那也就说必须保证这个配置文件对象的唯一性。

怎样保证对象的唯一性？

  • 一个类只要提供了构造函数，就可以产生多个对象，完全无法保证对象唯一。既然数量不可控，干脆不让其他程序建立对象
  • 不让其他程序创建，对象何在？干脆自己再本类中创建一个对象，这样就可控了。
  • 创建完成后，需要给其他程序提供访问的方式

实现步骤

  • 不让其他程序创建对象？直接私有化构造函数，不让其他程序创建的对象初始化。
  • 直接在本类中new一个本类的对象
  • 定义一个功能，其他程序可以通过这个功能获取到本类的对象
