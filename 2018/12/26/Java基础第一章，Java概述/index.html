1.1 什么是计算机语言

1.2 Java语言概述

  • 是SUN(Stanford University Network，斯坦福大学网络公司)1995年推出的一门高级编程语言。
  • 是一种面向Internet的编程语言。
  • 随着Java技术在web方面的不断成熟，已经成为Web应用程序的首选开发语言。
  • 是简单易学，完全面向对象，安全可靠，与平台无关的编程语言。

1.2.1 Java语言的三种技术架构

  1. JAVAEE（Enterprise Edition）：开发企业环境，针对Web应用程序开发
  1. JAVASE（Standard Edition）：开发普通桌面和商务应用程序。是其他两者的基础，可完成一些桌面应用程序的开发。
  1. JAVAME（Micro Edition）：开发电子消费产品和嵌入式设备。用于小型电子消费类产品，如手机应用程序。

1.2.2 Java语言的特点：跨平台性

跨平台性：通过Java语言编写的应用程序在不同的系统平台上都可以运行

原理：能跨平台是因为Java有自己的虚拟机(JVM Java Virtual Machine)，先安装JVM，由JVM来负责Java程序在不同平台运行。因为虚拟机可能会调用系统底层的东西，所以不同系统都对应不同版本的JVM，但是不需要改变Java程序

1.3 Java语言的环境搭建

1.3.1 什么是JRE和JDK？

JRE(Java Runtime Environment)：运行环境；包括JVM和Java程序所需的核心类库等，如果想要 运行 一个开发好的Java程序，计算机中只需要安装JRE即可

JDK(Java Development Kit)：开放工具包；JDK是提供给Java开发人员使用的，其中包含了java的开发工具，也包括了JRE。所以安装了JDK，就不用在单独安装JRE了。

1.3.2 JDK下载

www.oracle.com或 java.sun.com

1.3.3 JDK安装

安装JDK时，会提示安装JRE，可以选择不安装。而且到一台新的机器上时，可以直接把JDK的目录考过去，不需要重新安装。只是安装会在注册表里有注册，联网下有更新会提示。就这么点区别。

1.3.4 配置环境变量path

原理：在DOS中，若当前执行的程序在当前目录下不存在，系统会在path的环境变量指定的目录中查找。这样只要path中有该程序的运行路径，该程序就可在任一目录下运行

配置：path环境变量下，将JDK的bin目录的完整路径复制到其中，并与前后的目录用；分隔开。也可以在系统变量中，令变量JAVA_HOME为bin之前的目录，然后再path中添加%JAVA_HOME%\bin；

1.4 Java程序开发体验

  1. 将java代码编写到扩展名为.java的文件中
  2. javac命令对.java文件进行编译，生成.class文件（错误是在编译时报出来的）
  3. java命令（调用JVM）对生成的.class文件运行（其实就是虚拟机调用你类中的主函数。你需要这个类可以独立运行才需要主函数，不需要它独立运行，这个类就可以不写主函数。主函数就是程序的入口）
  4. 示例
    • 新建一个txt文件，写好小程序后保存成demo.java文件
    • dos进入该文件的目录下编译，javac demo.java（后缀名不能省）
    • 运行：java demo（后缀名可省）

1.5 配置classpath

classpath变量值是java类文件的目录：只找classpath中的路径

  • 值的结尾处有分号，那么JVM在classpath目录下没有找到要指定的类文件，会在当前目录下在查找一次。
  • 值的结尾处没有分号，那么JVM在classpath目录下没有找到要指定的类文件，不会在当前目录下查找。
  • 如果需要将当前路径添加到classpath中，不要用分号，建议 classpath=.;某一目录;某二目录。其中.表示当前路径

path变量是windows可执行程序文件的目录：先在当前目录下查找，再找path环境变量中的路径

1.6 创建问题及解决方法

固定格式：类名由一个或多个单词组成，每个单词首字母大写

主函数：public static void main(String[] args)

注释

  • 单行注释//，多行注释/ /，除多行注释不能嵌套多行注释，其他可以
  • 文档注释/* /，写在代码最前面，说明作者、时间、代码输入输出信息等
  • 写代码过程中必须加注释，并且每行都加注释导致阅读性很差，可以几行难懂的程序集体加一行注释；
  • 注释也用来调试程序：把怀疑有错的代码注释掉，编译不报错的话，则怀疑正确~
