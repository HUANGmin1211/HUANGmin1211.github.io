Java基础第四章 继承

Posted on 2019-01-02 | In Java | 浏览 次

原创，转载请注明出处

概述

引出：多个类中存在相同属性和行为时，将这些共性的内容抽取到单独一个类中，那么多个类无需再定义这些属性和行为，只要继那个类即可。

通过 extends 关键字让类与类之间产生继承关系。继承的出现提高了代码的复用性，让类与类之间产生了关系，提供了多态的前提

class SubDemo extends Demo{}

特点

什么时候使用继承？

  • 不要仅为了获取其他类中某个功能而去继承
  • 类与类之间要有所属( “ is a “ )关系，xx1是xx2的一种。

Java只支持单继承，不直接支持多继承（一个子类只能有一个父类）。因为若子类多继承的两个父类中具备了同名的方法，那子类调用该名方法时就会产生不确定性。但是Java支持多层继承(继承体系)

class A{}；
class B extends A{}；
class C extends B{}；

学习继承体系时，应该参阅顶层的类中的内容，了解这个体系的基本功能。使用这个体系功能时，需要创建最子类的对象

子父类成员特点

成员变量

子类不能直接访问父类中的私有的属性和行为，但是继承了，在子类对象的堆内存中是有父类的私有内容的

子父类定义了同名的成员变量num，都存在于子类的对象的堆内存中。子类对象直接访问num，则相当于this.num,访问的是子类的变量。想要访问父类的变量，用super.num。

成员函数

子父类中定义的同名的成员函数show()，调用时调用的是子类的函数。称为函数的override（重写，重载）

构造函数

当子父类都有构造函数时，先执行父构造函数，再执行子构造函数。因为子类所有的构造函数的第一行都有一句隐式的super();，所以子类中所有的构造函数默认都会访问父类中空参数的构造函数

当父类中没有空参数的构造函数时，子类的构造函数必须通过显式的super或this语句指定要访问的的构造函数

为什么子类构造函数中有一句隐式的super()？因为子类会继承父类中的内容，所以子类在初始化时，必须先执行父类的初始化动作，才能使用父类中的内容

如果子类的构造函数第一行显示的写了this调用本类其他构造函数，那super调用父类的语句就没有了。因为this和super只能定义在构造函数第一行，初始化动作要先执行。（本类其他构造函数默认仍调用super()）

父类构造函数中也有隐式的super()。只要是构造函数，默认第一行都是super()。父类的super()调用的谁的构造函数？Java体系在设计时，定义了一个所有对象的父类Object。即所有的类都隐式继承Object类

函数覆盖

定义：子类中出现与父类一模一样（名称、返回值、参数列表都一样）的方法时，会出现覆盖操作，也称为重写或者复写

注意

  • 覆盖时，子类方法权限一定要大于等于父类方法权限
  • 静态只能覆盖静态。其中父子任意一个为非静态都不行

应用

  • 当父类的部分功能是我需要的，部分功能不适合我，那我可以对不适合的部分进行复写，复写成我需要的功能
  • 在对老代码进行维护升级时，新功能单独描述单独封装。不要修改老代码，而且新建一个子类去继承老的类，获取父类的功能，并在子类中对需要升级的功能进行复写。
  • 若子类仅需对父类某函数添加新的功能，可以在复写的该函数里调用父类的函数super.函数名，再在后面添加新功能。

super和final关键字

super和this的用法相像

  • this代表本类对象的引用，可以单独使用
  • super代表父类的内存空间，必须带着.使用父类的成员
  • 当子父类出现同名成员时，可以用super进行区分

final关键字

  • 类：final修饰的类不可以被继承。
  • 方法：final修饰的方法不可以被覆盖。
  • 变量（局部变量，成员变量，静态变量）：final修饰的变量是一个常量，定义同时需赋值，只能被赋值一次。

当程序中的一个数据使用时是固定不变的，这时为了增加阅读星，可以给该数据起个名字，这就是变量。为了保证这个变量的值不能修改，加上final修饰，这就是一个阅读性很强的常量。写书写规范上，被final修饰的常量名所有字母都大写，单词间以下划线连接

抽象类

概述

抽象方法的由来：

多个对象都具备相同的功能，但是功能具体内容有所不同，那么在抽取过程中，只抽取了功能定义，并未抽取功能主体，那么只有功能声明，没有功能主体的方法称为抽象方法。

例如：狼和狗都有吼叫的方法，可是吼叫内容是不一样的。所以抽象出来的犬科虽然有吼叫功能，但是并不明确吼叫的细节。

abstract class 犬科{
    abstract void 吼叫();    //抽象函数
}

抽象方法与抽象类

Java中可以定义没有方法体的方法，该方法的具体实现由子类完成，该方法称为抽象方法，包含抽象方法的类就是抽象类。抽象类和抽象方法都要用abstract关键字修饰。

特点

  • 抽象方法只有方法声明，没有方法体，定义在抽象类中。

    格式：修饰符 abstract 返回值类型  函数名(参数列表) ;

  • 抽象类不可以被实例化，也就是不可以用new创建对象。因为调用抽象方法也没有意义。

  • 子类需要覆盖掉抽象类中所有的抽象方法后才可以创建对象，否则该子类还是抽象类。

相关问题

  • 抽象类一定是父类？是的。因为抽象类是不断向上抽取而来，而且抽象类只有被继承，才会有意义
  • 抽象类中是否有构造函数？有的。虽然不能自己创建对象，但是可以给子类的对象初始化。
  • 抽象类中可不可以没有抽象方法？可以。这个抽象类的意义仅是不让该类创建对象。见图没有抽象方法的抽象类
  • abstract不可以和哪些关键字共存？
    • final：final修饰不能被继承。抽象类需要被继承
    • private：抽象方法需要被子类覆盖，private不允许子类访问
    • static：static可以被类名调用，但是类名调用static的抽象方法没意义

抽象类和一般类的异同点

相同：都是用来描述事物；都可以定义属性和行为，包括构造函数
不同

  • 一般类可以具体的描述事物，抽象类描述事物不具体
  • 抽象类看可以多定义一种成员：抽象函数
  • 一般类可以创建对象，但是抽象类不能创建对象


接口

定义

当抽象类中的方法全是抽象的，可以通过接口的形式来体现。接口就是定义功能的，但是不定义功能体。谁需要这个功能谁去实现（个人理解，接口像是声明，声明这个类中应该有什么函数，但是并没有函数主体）

Interface Demo{ }//定义一个名为Demo的接口

定义格式

  • 定义变量，但是变量必须有固定的修饰符修饰：public static final，所以接口中的变量也称之为常量
  • 定义方法，但是可有固定的修饰符：public abstract

如果我不写这些修饰符，编译器会帮我加上的~

在接口的实现中，复写接口的函数一定要加上public，默认权限的话会报错，因为接口中方法的权限就是public，就算省略不写，也是public

特点

  • 不可以创建对象
  • 子类必须覆盖接口中所有的抽象方法后，子类才可以实例化，否则子类还是一个抽象类
  • 定义子类去覆盖接口中的方法。类与类的关系是继承，类与接口之间的关系是实现【implements】

接口的出现将解决多继承的弊端（当多个父类中有同名功能时，子类调用会产生不确定性），将“多继承”通过另一种形式体现出来，即“多实现”。其实因为多继承时，父类中的功能有主体，而导致调用运行时运行哪个主体内容。而多实现的接口中的功能都没有方法体，有子类来明确。

interface A{        
    void show();            //省略public,由编译器添加        
}        
interface B{
    void show()        
}

class c implements A,B{             //多实现        
    public void show(){ 方法体 };        
}

基于接口的扩展

引入：子类通过继承父类来扩展功能，但是通过继承而扩展的功能都是子类应该具备的基础功能。如果子类想要继续扩展其他类中的功能，此时应该通过实现接口来完成

因此接口的出现避免了单继承的局限性。继承的同时多实现代码

class Fu{
    void show() {}    
}

interface Inter{    
    void show1();    
}

class Zi extends Fu implements Inter{    
    public void show1() {}        //一定要写public    
}

抽象类中定义的都是事物的基本功能，接口中定义的都是事物的扩展功能。

接口出现后的一些小细节

类与类的关系是继承（is a），类与接口之间的关系是实现（like a）【inplements】

接口与接口之间是继承关系，而且可以多继承（用的不多）。代码：

interface Inter1{    
    void show1();    
}    
interface Inter2 {    
    void show2();    
}

interface Inter3 extends Inter1,Inter2{    
    void show3();    
}    

class Demo implements Inter3{    
    public void show2() {}    
    public void show1() {}    
    public void show3() {}    
}

接口的思想

接口的出现对功能实现了扩展；定义了规则；降低了耦合性（解耦）

接口的出现完成了解耦，说明有两方，一方再使用这个规则，另一方在实现这个规则。比如笔记本的USB口在使用规则，外围设备实现规则

接口和抽象类的区别

  • 抽象类描述事物的基本功能，可以定义非抽象方法；接口中定义事物的扩展功能，只能是抽象方法
  • 类与类的关系是继承（is a），类与接口之间的关系是实现（like a）
