Java基础第五章，多态

Posted on 2019-01-05 | In Java

原创，转载请注明出处

体现&好处&弊端&前提

体现：父类的引用或者接口的引用指向了自己的子类

Dog dog = new Dog();        //dog对象的类型是Dog类型 
Animal dog1 = new Dog();    //dog1对象的类型是Animal类型

好处：提高程序扩展性

弊端：通过父类引用操作子类对象时，只能使用父类中已有的方法，不能操作子类特有的方法

前提：必须有关系（继承，实现）；子类通常都要重写父类的方法，因为不同子类对父类中定义的相同函数有不同的函数体

转型

向上转型

Animal是父类型，new Dog()是子对象。当父类型引用指向子类对象时，就让子类对象进行类型的提升，即向上转型

Animal a = new Dog();
  • 好处：提高扩展性，隐藏子类型
  • 弊端：不能使用子类型的特有方法
  • 解决方法：向下转型（类型强制转换）

向下转型

Animal a = new Dog();  
Dog d = (Dog) a;  //此时就可以使用子类的特有内容

注意事项

向下转型因为不明确具体子类对象类型，所以容易引发ClassCaseException异常。为了避免这个问题，需要在向下转型前，做类型判断。判断类型用的是 instanceof

if( a instanceof Cat ){ 
    Cat c = (Cat) a;  
    c.catchMouse(); 
} //如果a指向的对象类型是Cat类型，再做类型转换

注意：无论向上还是向下转型，最终都是子类对象（new Dog()）在做着类型的变化。不能把父对象强制转换成子类型！

转型总结

  • 什么时候使用向上转型？提高程序的扩展性，不关心子类型（不需要子类的特有方法时）
  • 什么时候使用向下转型？要使用子类的特有方法时。但是一定要使用instanseof进行判断，避免发生ClassCaseException

多态成员调用的特点

成员变量：编译运行都看左边

  • 编译时期：参考引用型变量所属的类中是否有该成员变量，没有则编译失败
  • 运行时期：也是调用引用型变量所属的类中的成员变量

成员函数：编译看左边，运行看右边

  • 编译：参考左边。左边类型有该方法，则编译通过；如果没有，编译失败
  • 运行：参考右边对象所属的类。
  • 小敏理解：方法依赖于堆中的new的对象，所以要看右边的对象

静态函数：编译和运行都看左边

  • 小敏理解：静态函数存放于静态方法区，且是被类调用的，所以要看左边的类型

对于成员函数，是动态绑定到对象上。静态函数是静态的绑定到类上

注意：真正开发时，静态方法是不会被多态调用的，因为多态是对象的多态，而静态方法不所属于对象，应该类调用。

Object类常用方法

Object类是Java中所有类的父类，所有类都隐式的继承Object类，定义了所有对象都具备的功能

API文档（Application Program Interface，应用程序接口）。网上随便一搜 Java API文档中文

equals方法

  • Object用于判断两个对象是否相等，等价于 == ，但是用equals方法代替 == 更能体现代码的封装性
  • 所有的类都是Object的子类，因此所有的类都可以复写equals方法，比如String复写了equals方法用于判断两个字符串内容是否相同
  • API中，public boolean equals ( Object obj ) //这个形参就是多态的体现

equals

  • Object本身有equals方法，所以我们自己的对象如果也需要判断相等的方法的话，直接复写Object类的equals方法就可以了，不要自己瞎起名
