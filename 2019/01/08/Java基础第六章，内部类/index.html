Java基础第六章，内部类

Posted on 2019-01-08 | In Java

原创，转载请注明出处

内部类

引出：当A类中的内容要被B类直接访问，而A类也需要创建B的对象，访问B的内容时，可以将B类定义到A类的内部。这样访问更为便捷。B就叫做内部类（内置类，嵌套类）

定义：将一个类定义在另一个类的里面，对里面那个类就称为内部类（内置类，嵌套类）。

内部类或内部接口的名字中都带点，XxxYyy.ZzzAaa

什么时候使用？当描述事物时，事物的内部还有事物，且这个内部的事物还在访问外部事物中的内容，这时就将这个事物通过内部类来描述

访问方式：

  • 内部类可以直接访问外部类中的所有成员，包含私有成员；
  • 外部类要想访问内部类中的成员，必须创建内部类的对象

题外话：以后基本不会自己开发内部类，看得懂就行了，有时会在API里面用到一些别人已经定义好的内部类

内部类的访问

访问内部类的非静态成员

Outer.Inner in = new Outer().new Inner();    
in.show();

因为现在Inner是Outer的一个成员，成员需要被其对象调用，如果Inner内还调用了外部类的对象，那更得先new外部类，再new内部类啦。因此需要先 new outer，再 new inner

访问静态内部类的非静态成员

Outer.Inner inner = new Outer.Inner();    
inner2.show();

静态内部类是随着外部类的加载而加载，因此可以把静态内部类理解为一个外部类。因此不需要创建Outer对象，直接创建内部类对象。但是内部类得类名还是Outer.Inner

访问静态内部类的静态成员

Outer.Inner.staticshow();

静态内部类随外部类加载，静态成员也随着类加载，所以可以用类名调用

静态注意事项

  • 如果想要在内部类中定义静态成员，内部类必须要是静态的
  • 静态内部类只能访问外部类的静态变量，因为加载时间的问题

  • 非静态内部类中不允许定义静态成员，仅允许在非静态内部类中定义静态常量， 

    static final  int haha = 3;

  • 非静态内部类不会随着外部类加载而加载，只有出现了【new Inner()】语句时，非静态内部类才加载

  • 不管是静态内部类还是局部内部类，都是一个类，编译都是单独的class文件


第21行，只有在Inner类里面定义inner才能如图定义，在其他地方定义见下图的main函数

内部类能直接访问外部类中的成员的原因：因为内部类中其实持有了外部类的引用【外部类.this】。对于静态内部类不持有【外部类.this】，而是直接使用【外部类名】。

局部内部类

之前都是将内部类定义在外部类的成员位置上，其实内部类也可以定义在外部类的成员方法里面，称为【局部内部类】

局部内部类不是成员，不能被【成员修饰符】修饰了，public\static\private。

局部内部类也是出现了new语句的时候才加载，用到了才加载

内部类定义在局部时，只能访问被final修饰的局部变量。因为编译生成的class中就直接操作的final的数值。不能访问非final的局部变量

内部类的延伸

内部类是可以继承或实现外部其他的类或接口的

好处在于：可以通过内部类的方式对类进行重写或接口进行实现。然后通过公共的方式对其内部类对象进行访问。因为通常内部类很有可能被外部类封装其中。我们就可以通过父类和接口的方式来访问内部类的对象。

匿名内部类

就是有名内部类的简化写法：带有内容的子类对象。一定是子类对象！因此万一向上转型成父类或者Objet类，就不能调用子类特有方法了

格式为：new 外部类名或者接口名 () { 覆盖类或者接口中的代码，也可以自定义内容}; 不要忘记分号！

前提：内部类必须继承或实现一个外部类或者接口。


简单理解：就是建立一个带内容的外部类或者接口的子类匿名对象。

匿名内部类中方法不要过多，不然阅读性会很差
